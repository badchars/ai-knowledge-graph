## Зачем этот проект

Революция искусственного интеллекта привнесла **сотни новых технических терминов** в наш лексикон всего за несколько последних лет. От _архитектуры трансформеров_ до _атак с инъекцией промптов_, от _форматов квантования_ до _фреймворков агентного ИИ_ — терминологический ландшафт расширяется быстрее, чем любой учебник, курс или документация могут за ним успеть.

**Это реальная проблема для всех технических специалистов:**

- **Разработчик**, интегрирующий API LLM, сталкивается с терминами вроде _KV-cache_, _temperature_, _top-p_, _system prompt_ — но как они связаны? Каковы последствия для безопасности каждого из них?
- **Специалист по кибербезопасности**, проводящий аудит системы ИИ, должен понимать _непрямую инъекцию промптов_, _отравление RAG_, _отказ в обслуживании модели_ — но где же карта?
- **DevSecOps-инженер**, развертывающий vLLM на Kubernetes, должен знать о _тензорном параллелизме_, _непрерывной пакетной обработке_, _PagedAttention_, _перехвате потоков CUDA_ — всё в контексте, всё связано.
- **Исследователю** необходимо видеть, как _дифференциальная конфиденциальность_ связана с _извлечением обучающих данных_, как _RLHF_ связан с _выравниванием_, как _MoE_ меняет ландшафт вывода.

**Существующие ресурсы разрознены.** Статьи в блогах охватывают отдельные темы. Документация существует в изоляции. Ни один ресурс не отображает связи между концепциями ИИ/МО, их последствиями для безопасности и практическими конфигурациями — на нескольких языках.

**Этот проект заполняет этот пробел.** Интерактивный граф знаний с силовым направлением, который визуализирует, как 132 концепции ИИ/МО связаны друг с другом, с подробным контентом об уязвимостях безопасности, конфигурациях защиты, научных статьях и практических советах — на 6 языках.

## Что содержит каждый термин

Каждый узел в графе открывает богатую, структурированную карточку знаний:

| Раздел | Содержимое | Пример (`RAG`) |
|---------|---------|-----------------|
| **Описание** | Объяснение на 2-3 абзаца с встроенными ссылками | Как работает генерация с расширенным поиском, ключевые компоненты, почему это уменьшает галлюцинации |
| **Аналогия** | Реальное сравнение для интуитивного понимания | _"Как экзамен с открытой книгой — модель ищет ответы вместо того, чтобы запоминать всё"_ |
| **Безопасность** | Уязвимости, специфичные для механизма, с уровнем критичности | Отравление документов (высокий), манипуляция поиском (высокий), инъекция контекста (средний) |
| **Конфигурация** | Рабочие примеры кода для защиты | Python: ограничения размера фрагментов, валидация эмбеддингов, фильтрация поиска |
| **Исследования** | Статьи, официальная документация, руководства OWASP | Ссылки на arXiv, документация поставщиков, рекомендации по безопасности |
| **Советы** | Краткие практические рекомендации | _"Всегда проверяйте полученные документы перед внедрением в промпт"_ |

<br/>

### Контент с приоритетом безопасности

Раздел безопасности каждого термина **привязан к его собственному механизму** — не общие советы.

| Термин | Фокус безопасности |
|------|---------------|
| **GPU** | Уязвимости изоляции памяти, атаки по сторонним каналам, перехват потоков CUDA |
| **Transformer** | Запоминание самовнимания, манипуляция позиционным кодированием |
| **KV-Cache** | Отравление кэша между арендаторами, утечка состояния внимания |
| **Tokenizer** | Атаки с омоглифами, контрабанда BPE-токенов, манипуляция Unicode |
| **RAG** | Отравление документов, манипуляция поиском, инъекция контекста |
| **LoRA** | Отравление адаптеров, злонамеренная инъекция адаптеров на общих хабах |
| **vLLM** | Несанкционированный доступ к API, кража модели через конечные точки обслуживания |
| **HuggingFace** | RCE через десериализацию Pickle, атаки на цепочку поставок моделей |
| **Prompt Injection** | Атаки на основе RAG, перехват использования инструментов, манипуляция агентами |
| **Chat Template** | Эксплуатация токенизатора, обход границ системного промпта |

## Дизайн

Граф использует **минималистичный визуальный язык в стиле Apple** под названием **Snow**:

- Чистые круги с низкой прозрачностью заливки категории и тонкой центральной точкой
- Квадратичные кривые Безье для связей — почти невидимые, подсвеченные при взаимодействии
- Семейство шрифтов `Inter` с интервалами и иерархией веса Apple HIG
- Светлая тема: чисто белый фон `#fff`, карточки поверхности `#f5f5f7`
- Темная тема: фон `#1c1c1e`, поверхности `#2c2c2e` — нативная темная палитра Apple
- Без градиентов, без эффектов свечения, без анимации узлов — чистая ясность

| Функция | Детали |
|---------|---------|
| **Тема** | Светлая и темная с плавным переходом |
| **Размер узла** | Пропорционален количеству связей |
| **Поведение связей** | Прозрачные по умолчанию, подсвечены при наведении/выборе |
| **Постоянная подсветка** | Выбранный узел сохраняет видимые связи, пока открыта панель |
| **Поиск в прожекторе** | `Cmd+K` / `Ctrl+K` для поиска по всем терминам |
| **Адаптивность** | Макеты для рабочего стола, планшета и мобильных устройств |

## Начало работы

### Предварительные требования

- [Bun](https://bun.sh) v1.0+

### Установка и сборка

```bash
git clone https://github.com/badchars/ai-knowledge-graph.git
cd ai-knowledge-graph
bun install
bun run build
```

### Локальный предварительный просмотр

```bash
bun run dev          # собирает и открывает в браузере
```

### Проверка данных

```bash
bun run validate     # проверяет все файлы YAML на соответствие схеме
```

## Развёртывание на GitHub Pages

### Автоматическое (рекомендуется)

Репозиторий включает рабочие процессы CI/CD, которые обрабатывают всё:

1. **Форкните** или отправьте этот репозиторий на свой аккаунт GitHub
2. Перейдите в **Settings > Pages**
3. В разделе "Build and deployment" выберите **Source: GitHub Actions**
4. Готово — каждый push в `main` запускает сборку и развертывание

```
push в main → bun install → build → загрузка dist/ → развертывание на Pages
```

### Вручную

```bash
bun run build
# Загрузите dist/ на любой статический хост:
# GitHub Pages, Netlify, Vercel, Cloudflare Pages и т.д.
```

### Проверка PR

Каждый pull request, затрагивающий `data/**`, автоматически запускает:
- `bun run validate` — проверка схемы YAML
- `bun run build` — гарантирует сборку сайта
- Если что-то не проходит, PR не может быть объединен

## Участие в проекте

Мы приветствуем вклад от всех — исследователей ИИ, специалистов по безопасности, разработчиков, студентов.

### Быстрый старт

```bash
# 1. Форкните и клонируйте
git clone https://github.com/ВАШ_ПОЛЬЗОВАТЕЛЬ/ai-knowledge-graph.git
cd ai-knowledge-graph && bun install

# 2. Создайте новый термин
cp data/terms/_template.yaml data/terms/my-new-term.yaml

# 3. Отредактируйте файл YAML

# 4. Добавьте связи в data/relationships.yaml

# 5. Проверьте и протестируйте
bun run validate && bun run build

# 6. Отправьте PR
```

### Типы вклада

| Тип | Сложность | Влияние |
|------|-----------|--------|
| Добавить новый термин | Средняя | Высокое |
| Добавить переводы | Легкая | Высокое |
| Исправить описания | Легкая | Среднее |
| Добавить находки по безопасности | Средняя | Очень высокое |
| Добавить ссылки на исследования | Легкая | Среднее |
| Добавить примеры конфигурации | Средняя | Высокое |
| Сообщить о неточностях | Легкая | Высокое |

### Формат файла термина

Каждый термин — это отдельный файл YAML в `data/terms/`:

```yaml
id: my-new-term              # должен совпадать с именем файла
name: MyTerm                  # короткое отображаемое имя
fullName: My Full Term Name   # расширенное имя
category: architecture        # одна из 11 категорий

desc:
  en: |
    English description with [inline links](https://example.com).
    Multiple paragraphs separated by blank lines.
  tr: |
    Turkish description...
  fr: ""
  zh: ""
  ar: ""
  ru: ""

analogy:
  en: "A real-world comparison..."

security:
  - title:
      en: Vulnerability Name
    description:
      en: How this vulnerability works.
    severity: high       # critical | high | medium | low

research:
  - url: https://arxiv.org/abs/xxxx.xxxxx
    label:
      en: Paper Title

tips:
  - en: A quick actionable tip
```

### Добавление связей

```yaml
# data/relationships.yaml
- source: my-new-term
  target: transformer
  label:
    en: uses
    tr: kullanir
    fr: utilise
    zh: 使用
    ar: يستخدم
    ru: использует
```

### Шаблоны issues

Код не требуется — используйте наши формы:

- **[Предложить новый термин](../../issues/new?template=new-term.yml)**
- **[Обновить термин](../../issues/new?template=update-term.yml)**
- **[Добавить перевод](../../issues/new?template=add-translation.yml)**

См. **[CONTRIBUTING.md](CONTRIBUTING.md)** для полного руководства.

## Для кого этот проект

| Роль | Как это помогает |
|------|---------------|
| **Разработчики** | Понимать термины ИИ/МО, с которыми вы сталкиваетесь ежедневно — что они означают, как связаны, последствия для безопасности |
| **Специалисты по безопасности** | Картографировать поверхность атаки ИИ — от инъекции промптов до кражи модели, с уязвимостями, специфичными для механизма |
| **DevSecOps-инженеры** | Руководства по защите с реальным кодом — каждая находка по безопасности включает примеры конфигурации |
| **Пентестеры** | Знание архитектуры раскрывает векторы атак — понимать системы ИИ достаточно глубоко, чтобы их тестировать |
| **Инженеры ИИ/МО** | Быстрая справка по экосистеме — инструменты, фреймворки и техники в контексте |
| **Студенты и исследователи** | Аналогии делают концепции интуитивными, ссылки на исследования указывают на первоисточники |
